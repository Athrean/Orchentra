Last refreshed: 2026-07-02.

## CommandCode

CommandCode is the direct commercial competitor to track first.

Sources:

- Landing: <https://commandcode.ai/>
- Features: <https://commandcode.ai/features>
- Pricing: <https://commandcode.ai/pricing>
- Provider API: <https://commandcode.ai/provider>
- Quickstart: <https://commandcode.ai/docs/quickstart>
- Pricing limits: <https://commandcode.ai/docs/resources/pricing-limits>

### What They Sell

- Terminal coding agent.
- `cmd` short binary.
- Taste learning from accept/reject/edit signals.
- Project-level skills and personal memory.
- Interactive CLI, headless mode, yolo/background sandbox.
- MCP, custom commands, plugins, and agents.
- Studio for usage, billing, taste packages, and org settings.
- Credit/subscription plans with provider discounts and top-ups.
- Team sharing through taste package push/pull.

### What We Should Steal

- Make learning visible in the product, not only in docs.
- Add first-class spine commands: `/terse`, `/budget`, `/lean`.
- Show savings and verification receipts in the TUI.
- Keep a short alias and self-update path sharp.
- Add usage/credit UI only if the hosted credit proxy is approved.

### Where We Differentiate

- Orchentra's core promise is efficiency plus verifiable review, not generic taste learning.
- `/review` must stay stronger by running checks and tying findings to evidence.
- BYOK/no-telemetry/no-account mode stays the default privacy position.
- Lean-code behavior should make diffs smaller, not merely more personalized.

## Claude Code

Claude Code remains the platform benchmark for IDE integrations, hooks, subagents, skills/plugins, and broad ecosystem expectations.

Do not compete by cloning every surface. Compete by being smaller, cheaper to run, and more verifiable.

## Kimi CLI

Kimi is the benchmark for distribution polish, single-binary install, session UX, ACP/IDE protocol work, web/trace visualization, and broad slash command ergonomics.

Steal selectively: `/undo`, `/fork`, `/add-dir`, session browser, and trace visualization are useful. Avoid platform sprawl until the spine is stronger.

## Harness-First Competitors

The strongest idea from harness-oriented competitors is deterministic scenario testing:

- mock provider
- scripted tool calls
- permission scenarios
- compaction/budget scenarios
- machine-readable receipts

Orchentra already has a scenario harness substrate. It needs broader coverage.
